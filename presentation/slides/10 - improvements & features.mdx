### Better reactivity
- Vue3 uses proxy objects for reactivity
- Objects are now fully reactive by default without any change detection issues
- Objects and Arrays can be made partially reactive
<!-- - Requires evergreen browsers (newer than IE11) -->
- Makes destructuring slightly more complicated

### Event-validation and event-documentation
<!-- block-start: grid -->
<!-- block-start: column -->
<!-- classes: grid -->
- Events can now be validated via the emits-property
- See at a glance which events are emitted by a component and check the validity of emitted event parameters
- Eslint shows warning when events that are used are missing in the emits property
- Can be simple string array for documentation only
- Can be an object of callback functions for validator functionality

### Teleport
<!-- block-start: grid -->
<!-- block-start: column -->
<!-- classes: grid -->
- New tag to break out of the current component hierarchy
- Elements are appended to the body tag by default and act as regular child elements (receive props etc.)
- Avoids common stacking context issues and containing block issues
- Mainly useful for overlays, toast messages etc.
- Multiple teleports to the same target will be appended in the order they are called
